What the agent does.

  1. 01Reads each new R&R nomination at submission and resolves the recipient, nominator, and project context from Workday.
  2. 02Decides — auto-approves when the policy checks come back clean (value, frequency, recipient eligibility, nominator legitimacy); otherwise routes the nomination for human review with a tagged reason.
  3. 03Hands each exception to the existing Workday BP reviewer with the specific reason it was flagged and the context the reviewer would otherwise have to look up.
  4. 04Watches for patterns the rule cascade alone can’t see — reciprocity, ring nominations, off-policy bursts (Phase 2).
  5. 05Writes every decision, with the agent’s reasoning, to a SharePoint Excel sheet / Workday worksheet for full team transparency.

What could go wrong, and how we contain it.

Conservative defaults at launch. People Ops keeps oversight; the agent never owns the policy.

Risk Mis-approves a non-policy nomination.
Mitigation Conservative thresholds at launch, weekly People Ops sample audit of the SharePoint sheet / Workday worksheet, full reversal in one click.
Risk Abuse patterns slip through the rules.
Mitigation Phase 2 detection layer; anomalies raised through the existing Workday BP routing and flagged on the SharePoint Excel sheet / Workday worksheet.
Risk Policy changes mid-cycle.
Mitigation Thresholds versioned and re-validated each refresh.

Three phases, each shippable on its own.

MVP
Policy-threshold auto-approve, exception routing via the existing Workday BP, SharePoint audit sheet / Workday worksheet.
Phase 2
Pattern-detection layer — the agent reads cross-nomination signals (reciprocity, frequency anomalies, off-policy values, gaming) the rule cascade alone can’t see.
Phase 3
Generalise the agent pattern across other rubber-stamp Workday business processes.

No Workday portfolio overlap blocks this — the Self-Service Agent only handles policy Q&A; it doesn’t approve.
